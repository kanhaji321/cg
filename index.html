(A) Define the following terms: (24 × ½ = 12 marks)

Ans:
1. ComputerThe computer is an information processing machine. It is a tool for storing, manipulating and correlating data.
2. GraphicsGraphics objects such as pictures, charts, graphs and diagrams used to communicate processed information instead of simple text.
3. Computer GraphicsThe pictorial synthesis of real or imaginary objects from their computer-based models.
4. GUIGraphical interfaces that use visual control items such as buttons, menus, icons, and scroll bars to allow user interaction via mouse-clicks.
5. SimulationThe use of graphics to make mathematical models and mechanical systems more realistic and easy to study.
6. Image ProcessingAlso called picture analysis, it concerns the analysis of scenes, or the reconstruction of models of 2D or 3D objects from their picture.
7. Virtual RealityA computer-generated immersive 3D environment that simulates real or imaginary worlds. Users interact with this environment using special devices like VR headsets, gloves, and motion sensors.
8. Snap ShotA single static image captured at a specific instant of time from a video, animation, or screen display. It represents the state of the visual output at that particular moment.
9. FrameA single complete image in a sequence of images that form an animation or video. Multiple frames displayed rapidly (e.g., 24-60 per second) create the illusion of smooth motion.
10. Frame BufferA special area of memory dedicated to graphics that holds the set of intensity values for all the screen points.
11. PixelThe smallest addressable screen element which we can control by setting its intensity and colour.
12. Video ClipA short continuous sequence of frames that forms a moving image or animation. It is stored digitally and can be played back at a specified frame rate to show motion.
13. Raster-Scan SystemsA display system where the electron beam is swept back and forth from left to right across the screen, turning on and off to create images.
14. PersistenceThe time it takes the emitted light from the screen to decay to one-tenth of its original intensity.
15. ResolutionThe maximum number of points that can be displayed without overlap on the CRT, defined as points per centimeter horizontally and vertically.
16. BrightnessThe perceived intensity of light emitted by a display screen or an individual pixel. It determines how bright or dim the image appears to the viewer and is controlled by pixel luminance values.
17. IntensityA numerical value (typically ranging from 0 to 255 for 8-bit systems) that represents the luminance or brightness level of a pixel. A value of 0 means black (no light) and 255 means full brightness.
18. INK JET PrinterA non-impact printer that places extremely small droplets of ink onto paper to create an image.
19. TonerThe oppositely charged ink powder used in laser printers and electrostatic plotters that attracts to charged exposed areas to form an image.
20. Refresh BufferThe display buffer memory used to store the computer-produced display list and used for refreshing the display to avoid flicker.
21. Anti-AliasingThe process of adjusting intensities of the pixels along the line to minimize the effect of aliasing (the jagged stair-step effect).
22. CMYRefers to the cyan, magenta, and yellow ink colours used in printers, which combine to create any colour in the visible spectrum.
23. HSVHue, Saturation, Value — a color model based on how humans actually perceive color. Hue represents the color type (0°-360°), Saturation is the purity/vividness, and Value is the brightness level.
24. JPG/JPEGA specific digital file format in which devices like scanners store captured image information.

(B) Transformation Matrix — 2D Rotation about Arbitrary Point (2 marks)

Ans:
  • Rotation about an arbitrary point $P(x_r, y_r)$ means rotating an object by some angle $\theta$ around a fixed point that is NOT the origin. This is a very common requirement in real-world graphics applications.
  • Since the standard rotation matrix only works about the origin (0,0), we need to use a 3-step composite transformation process to achieve this rotation correctly.

Steps:

  • Translate the arbitrary pivot point $P(x_r, y_r)$ to the origin by applying $T(-x_r, -y_r)$. This shifts the entire object so that the pivot point sits at $(0,0)$.
  • Rotate the object by the desired angle $\theta$ about the origin using the standard rotation matrix $R(\theta)$. Now the rotation works correctly since the pivot is at origin.
  • Translate back to the original position by applying $T(x_r, y_r)$. This moves the object and the pivot point back to where they originally were.
Step 1: Translate to origin: $$T(-x_r, -y_r) = \begin{bmatrix} 1 & 0 & -x_r \\ 0 & 1 & -y_r \\ 0 & 0 & 1 \end{bmatrix}$$ Step 2: Rotate by $\theta$ about origin: $$R(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta & 0 \\ \sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{bmatrix}$$ Step 3: Translate back: $$T(x_r, y_r) = \begin{bmatrix} 1 & 0 & x_r \\ 0 & 1 & y_r \\ 0 & 0 & 1 \end{bmatrix}$$
Final Transformation Matrix:
$T(x_r, y_r) \cdot R(\theta) \cdot T(-x_r, -y_r)$
Combined Result: $$\begin{bmatrix} \cos\theta & -\sin\theta & x_r(1-\cos\theta) + y_r\sin\theta \\ \sin\theta & \cos\theta & y_r(1-\cos\theta) - x_r\sin\theta \\ 0 & 0 & 1 \end{bmatrix}$$

(C) Working of Laser Printer — Point Wise (All 07 Points) (2 marks)

Ans:
  • Data Sent: The document data is sent from the computer to the printer's internal processor as digital signals, which are then converted into a page image.
  • Laser Scans Drum: A focused laser beam scans across a rotating photosensitive drum surface, line by line, creating a pattern based on the image data.
  • Electrostatic Image Formed: The laser discharges selected areas on the drum surface, forming an invisible electrostatic (latent) image that corresponds to the text or graphics being printed.
  • Toner Attracted: Fine, electrically charged toner powder is applied to the drum, and it sticks only to the areas that were discharged by the laser beam.
  • Paper Passes Near Drum: A sheet of paper is fed through the printer and given an opposite electrical charge so it can attract the toner from the drum surface.
  • Toner Transfers to Paper: The toner particles transfer from the drum onto the paper surface due to the electrostatic attraction between the charged paper and the toner.
  • Heat Fuses Toner: Finally, the fuser unit applies both heat and pressure to permanently melt and bond the toner particles onto the paper, producing the final printed output.

(D) Construction & Working of LAP-TOP Screen — Point Wise (2 marks)

Ans:
  • LCD Panel: This is the main display layer of the laptop screen. It controls the amount of light passing through each pixel position to form the visible image on screen.
  • Backlight (LED): An array of LED lights placed behind the LCD panel that provides a uniform white light source. Without this backlight, the LCD panel alone cannot produce any visible image.
  • Polarizing Filters: Two polarizing filters are placed on either side of the liquid crystal layer. They regulate the direction of light waves before and after passing through the crystals to control brightness.
  • Liquid Crystal Layer: This layer contains liquid crystal molecules that twist or untwist when an electrical voltage is applied, which controls the intensity of light passing through each individual pixel.
  • Color Filters (RGB): A layer of tiny Red, Green, and Blue color filters placed over each sub-pixel. By combining these three primary colors in varying intensities, the display can reproduce millions of different colors.
  • Glass Substrates: Two thin, transparent glass plates that sandwich and hold the liquid crystal material in place. They also provide structural support and proper alignment for the crystal molecules.
  • Driver Circuit: An integrated circuit board that supplies precise electrical signals to control each individual pixel on the screen. It receives data from the GPU and converts it into pixel-level voltage signals.

Q.No.3 — Transformations & Coordinate Systems 10 MARKS

(A) Why exactly we need Homogeneous Coordinate System? (2 marks)

Ans:
  • In standard 2D Cartesian coordinates, Translation is handled by matrix addition, while Rotation and Scaling are handled by matrix multiplication.
  • This inconsistency prevents combining multiple transformations into a single matrix.
  • By introducing a third coordinate ($w = 1$), the Homogeneous Coordinate System converts translation into a matrix multiplication using a $3 \times 3$ matrix.
  • This allows any sequence of transformations — translation, rotation, scaling — to be concatenated into a single composite matrix, reducing computational overhead.

(B) Two successive 2D Rotations about origin are Commutative — Prove (3 marks)

Ans:
  • Claim: Two rotations are commutative: $R(\theta_1) \cdot R(\theta_2) = R(\theta_2) \cdot R(\theta_1)$
Rotation matrices: $$R(\theta_1) = \begin{bmatrix} \cos\theta_1 & -\sin\theta_1 \\ \sin\theta_1 & \cos\theta_1 \end{bmatrix}, \quad R(\theta_2) = \begin{bmatrix} \cos\theta_2 & -\sin\theta_2 \\ \sin\theta_2 & \cos\theta_2 \end{bmatrix}$$ Computing $R(\theta_1) \cdot R(\theta_2)$: $$= \begin{bmatrix} \cos\theta_1\cos\theta_2 - \sin\theta_1\sin\theta_2 & -(\cos\theta_1\sin\theta_2 + \sin\theta_1\cos\theta_2) \\ \sin\theta_1\cos\theta_2 + \cos\theta_1\sin\theta_2 & \cos\theta_1\cos\theta_2 - \sin\theta_1\sin\theta_2 \end{bmatrix}$$ Using trig identities: $$R(\theta_1) \cdot R(\theta_2) = \begin{bmatrix} \cos(\theta_1 + \theta_2) & -\sin(\theta_1 + \theta_2) \\ \sin(\theta_1 + \theta_2) & \cos(\theta_1 + \theta_2) \end{bmatrix} = R(\theta_1 + \theta_2)$$
  • Similarly, $R(\theta_2) \cdot R(\theta_1) = R(\theta_2 + \theta_1) = R(\theta_1 + \theta_2)$ — same result.
$\therefore R(\theta_1) \cdot R(\theta_2) = R(\theta_2) \cdot R(\theta_1) = R(\theta_1 + \theta_2)$ → Rotation about origin is COMMUTATIVE ✓

(C) Reflection in y=x and y=−x using Scaling + Rotation (2 marks)

Ans:

1. Reflection about y = x:

  • Standard reflection matrix:
$$\begin{bmatrix} 0 & 1 \\ 1 & 0 \end{bmatrix}$$ Using Rotation + Scaling: $$R(90°) \cdot S_{x\text{-axis}} = \begin{bmatrix} 0 & -1 \\ 1 & 0 \end{bmatrix} \begin{bmatrix} 1 & 0 \\ 0 & -1 \end{bmatrix} = \begin{bmatrix} 0 & 1 \\ 1 & 0 \end{bmatrix}$$

2. Reflection about y = −x:

  • Standard reflection matrix:
$$\begin{bmatrix} 0 & -1 \\ -1 & 0 \end{bmatrix}$$ Using Rotation + Scaling: $$R(90°) \cdot S_{y\text{-axis}} = \begin{bmatrix} 0 & -1 \\ 1 & 0 \end{bmatrix} \begin{bmatrix} -1 & 0 \\ 0 & 1 \end{bmatrix} = \begin{bmatrix} 0 & -1 \\ -1 & 0 \end{bmatrix}$$

(D) Show how Shear Transformation can be expressed in terms of Rotation & Scaling (3 marks)

Ans:
  • Any general shear matrix can be expressed as:
$$\text{Shear} = R(\theta) \cdot S \cdot R(-\theta)$$ Steps:
1. Rotate the object by $\theta$ to align axes: $R(\theta)$
2. Apply non-uniform scaling along these axes: $S$
3. Rotate back by $-\theta$: $R(-\theta)$
Conclusion: This shows that shear is a combination of standard rotation and scaling, not a fundamentally independent operation.
Q.No.4 — Visible Surface Detection 10 MARKS

(A) Image Space & Object Space Algorithm — Back Face Removal (1 mark)

Ans:
Object Space Algorithm:
Comparisons and visibility decisions are done between 3D objects in their original world coordinates BEFORE projection onto the 2D screen. These methods work with the actual geometry. Examples: Back-face removal, BSP Tree method.
Image Space Algorithm:
Visibility decisions are made at the individual pixel level AFTER all objects have been projected onto the 2D screen. These methods process the final image pixel by pixel. Examples: Z-Buffer, A-Buffer, Ray Casting.

(B) A-Buffer Algorithm — Point Wise, Why it's Expansion of Z-Buffer, Limitations (4 marks)

Ans:

A-Buffer Algorithm (Point Wise):

  • Each pixel position in the buffer contains a linked list of surface fragments instead of storing just a single depth value like the Z-Buffer does.
  • Each fragment in the list stores multiple pieces of data: depth (z-value), opacity (α), RGB color values, a coverage mask for anti-aliasing, and the surface ID it belongs to.
  • For each polygon in the scene, and for each pixel that polygon covers, the algorithm computes the fragment data and adds it to that pixel's linked list.
  • After all polygons in the scene have been processed, the fragments at each pixel are sorted by their depth values from front to back.
  • The sorted fragments are then composited together using alpha blending formula: C = α·C_front + (1−α)·C_back, which correctly handles transparency.
  • The final composited color value is then written to the frame buffer and displayed on screen at that pixel location.

Why called Expansion of Z-Buffer?

  • The Z-Buffer algorithm stores only ONE depth value per pixel in the depth buffer, which means it can only track the nearest surface and cannot handle transparent or semi-transparent objects at all.
  • The A-Buffer EXPANDS upon this limitation by storing a complete linked LIST of fragments per pixel, which allows it to properly handle multiple transparent and semi-transparent surfaces overlapping at the same pixel.
  • The "A" in A-Buffer stands for Anti-aliased, Area-averaged, Accumulation Buffer, reflecting its ability to produce smoother, higher quality rendered images.

Limitations of A-Buffer:

  • Requires very high memory usage because it must store multiple fragments (each with color, depth, opacity data) per pixel position.
  • Complex implementation compared to the simpler Z-Buffer, as it involves managing dynamic linked lists, sorting, and compositing operations.
  • Rendering is significantly slower because of the overhead of sorting fragment lists and performing alpha blending compositing at each pixel.
  • Not suitable for real-time rendering applications or systems with limited computational resources due to its heavy processing requirements.

(C) Painters Algorithm — Point Wise & Limitations (3 marks)

Ans:
  • Sort all polygons in scene by their depth (z-value) from farthest to nearest.
  • Paint (render) polygons starting from the farthest (back) to nearest (front).
  • Nearer polygons are painted OVER farther ones — like a painter painting background first.
  • Continue until all polygons rendered → final image displayed.
FIG: PAINTERS ALGORITHM — BACK TO FRONT RENDERING

Limitations:

  • Cyclic overlap — if A overlaps B overlaps C overlaps A, sort order cannot be determined.
  • Penetrating polygons — polygons that intersect each other cannot be handled correctly.
  • Sorting is expensive — O(n log n) time complexity.
  • No support for transparency.

(D) Construction & Working of Optical Mouse — Point Wise (2 marks)

Ans:
  • LED Illumination: An LED (Light Emitting Diode) at the bottom of the mouse illuminates the surface beneath it with a bright light, making surface texture details visible to the sensor.
  • CMOS Sensor: A tiny CMOS image sensor continuously captures hundreds of microscopic images of the illuminated surface texture every second.
  • DSP Processing: A built-in Digital Signal Processor (DSP) chip analyzes and compares successive captured surface images to detect any changes in position or pattern.
  • Movement Detection: By analyzing the differences between consecutive images, the DSP calculates the exact direction and speed of the mouse movement across the surface.
  • Data Transmission: The calculated movement data (dx, dy coordinates) is transmitted to the computer via USB cable or wirelessly through Bluetooth or RF signals.
  • Cursor Update: The computer's operating system receives the movement data and updates the cursor position on the screen accordingly, providing real-time visual feedback to the user.
Q.No.5 — Cohen-Sutherland + Transformation Table 10 MARKS

(A) Cohen-Sutherland Line Clipping Algorithm — Point Wise (5 marks)

FIG: COHEN-SUTHERLAND LINE CLIPPING
  • Step 1: Read two end points of the line say $P_1(x_1, y_1)$ and $P_2(x_2, y_2)$.
  • Step 2: Read two corners (left-top and right-bottom) of the window, say $(Wx_1, Wy_1)$ and $(Wx_2, Wy_2)$.
  • Step 3: Assign the region codes for two endpoints $P_1$ and $P_2$ using following steps:
    Initialize code with bits 0000
    Set Bit 1 – if $(x < Wx_1)$
    Set Bit 2 – if $(x > Wx_2)$
    Set Bit 3 – if $(y < Wy_2)$
    Set Bit 4 – if $(y > Wy_1)$
  • Step 4: Check for visibility of line $P_1 P_2$:
    (a) If region codes for both endpoints $P_1$ and $P_2$ are zero then the line is completely visible. Hence draw the line and go to step 9.
    (b) If region codes for endpoints are not zero and the logical ANDing of them is also nonzero then the line is completely invisible, so reject the line and go to step 9.
    (c) If region codes for two endpoints do not satisfy the conditions in 4a) and 4b) then the line is partially visible.
  • Step 5: Determine the intersecting edge of the clipping window by inspecting the region codes of two endpoints.
    (a) If region codes for both the end points are non-zero, find intersection points $P_1'$ and $P_2'$ with boundary edges of clipping window with respect to point $P_1$ and point $P_2$, respectively.
    (b) If region code for any one end point is non zero then find intersection point $P_1'$ or $P_2'$ with the boundary edge of the clipping window with respect to it.
  • Step 6: Divide the line segments considering intersection points.
  • Step 7: Reject the line segment if any one end point of it appears outside the clipping window.
  • Step 8: Draw the remaining line segments.
  • Step 9: Stop.

Q.No.6 — Short Notes (Any 5 × 2 marks = 10 marks) 10 MARKS

(A) Working of BAR Code Reader (2 marks)

Ans:
  • A barcode consists of a series of alternating dark bars and white spaces of varying widths that represent encoded data in a machine-readable format.
  • A laser beam or LED light source inside the barcode reader illuminates the barcode printed on the product or label.
  • A photodetector sensor inside the reader detects the reflected light — the dark bars absorb the light while the white spaces reflect it back to the sensor.
  • The pattern of reflected light is converted into electrical pulses that represent binary data (0s for dark bars and 1s for white spaces).
  • A built-in decoder circuit decodes the binary pulse pattern into human-readable data such as product numbers, prices, or identification codes.
  • The decoded data is then sent to the connected computer or POS (Point of Sale) system for processing, inventory management, or billing purposes.

(B) WARNOCK's Algorithm (2 marks)

Ans:
  • Warnock's algorithm is an area-subdivision based hidden surface removal algorithm that works by recursively dividing the screen into smaller regions.
  • The algorithm begins by dividing the entire screen into 4 equal quadrants (sub-areas), and then analyzes each quadrant independently.
  • For each area, the algorithm checks three cases: (a) If no polygon is present in the area → fill with background color. (b) If only 1 polygon completely covers the area → fill with that polygon's color. (c) If multiple polygons overlap in the area → subdivide that area further into 4 smaller quadrants.
  • This subdivision process continues recursively until either a simple case is found (no overlap or single polygon) or the area has been reduced to a single pixel.
  • The algorithm is very efficient for scenes that have few overlapping polygons, as most areas are resolved quickly without deep recursion.
  • Main limitation: It becomes very expensive and slow for complex scenes with many overlapping polygons, as the recursion depth can become extremely high.

(C) Z-Buffer Algorithm (2 marks)

Ans:
  • The algorithm maintains a Z-Buffer (also called depth buffer) which is a 2D array of the same size as the screen resolution, storing one depth value per pixel.
  • At the start, the Z-Buffer is initialized with the maximum depth value (∞, representing the farthest possible distance) and the Frame Buffer is filled with the background color.
  • For each polygon in the scene, and for each pixel (x,y) that the polygon covers, the algorithm calculates the depth z of the polygon at that pixel position.
  • If the calculated z is less than ZBuffer[x,y], it means this surface is closer to the viewer, so we update both: ZBuffer[x,y] = z and FrameBuffer[x,y] = polygon's color at that point.
  • If z is greater than or equal to ZBuffer[x,y], it means there is already a closer surface stored at that pixel, so we skip this polygon for that pixel.
  • Limitation: The Z-Buffer algorithm cannot handle transparent surfaces (since it only stores one depth per pixel), and it requires a large amount of memory equal to the frame buffer size.

(D) Inkjet Printer (2 marks)

Ans:
  • The printhead is mounted on a movable carriage that moves horizontally across the paper from left to right (and back), printing one line at a time.
  • The printhead contains hundreds of tiny nozzles (microscopic openings) through which extremely fine droplets of ink are sprayed onto the paper surface.
  • Thermal type (HP, Canon): A tiny heater element rapidly vaporizes the ink, creating a bubble that pushes and ejects a droplet through the nozzle onto the paper.
  • Piezoelectric type (Epson): A piezoelectric crystal deforms when voltage is applied, creating pressure that pushes an ink droplet out through the nozzle with precise control.
  • Color inkjet printers use CMYK (Cyan, Magenta, Yellow, and Black) ink cartridges. By mixing these four colors in varying amounts, the printer can produce a full range of colors.
  • After each horizontal pass, the paper advances by a small amount (one line), and the process repeats. The complete image is formed row by row from top to bottom of the page.

(E) Working of Scanner (2 marks)

Ans:
  • The document to be scanned is placed face-down on a transparent glass plate (called the platen) at the top of the scanner.
  • A bright light source (usually a fluorescent lamp or LED array) beneath the glass illuminates one narrow line of the document at a time.
  • The reflected light from the document passes through a system of lenses and mirrors which focus it onto a CCD (Charge Coupled Device) sensor array.
  • The CCD sensor converts the light intensity into analog electrical signals, where brighter areas produce stronger signals and darker areas produce weaker signals.
  • An ADC (Analog to Digital Converter) then converts these analog signals into digital values (binary numbers) representing the brightness and color of each pixel.
  • The scan head assembly slowly moves across the document from top to bottom, scanning the entire document line by line. The final digital image data is then stored in the computer's memory.

(G) RGB Color Model (2 marks)

Ans:
  • RGB is an additive color model, which means colors are created by adding (combining) different wavelengths of light together. Adding all three gives white light.
  • The three primary colors used are: Red (R), Green (G), and Blue (B). By mixing these three in different proportions, any visible color can be reproduced on screen.
  • Each color component has a value range of 0 to 255 (8 bits per channel), giving a total of 24 bits per pixel for full color representation.
  • Some common examples: White = (255, 255, 255), Black = (0, 0, 0), Pure Red = (255, 0, 0), Yellow = (255, 255, 0), and Cyan = (0, 255, 255).
  • Total number of possible colors = 2²⁴ = 16,777,216 ≈ 16.7 million colors, which is more than enough to represent photorealistic images on digital displays.
  • The RGB model is used extensively in: computer monitors, TV screens, digital cameras, mobile phone displays, and all light-emitting digital display devices.

(H) Top 5 Applications of Computer Graphics (2 marks)

Ans:
  • Entertainment & Animation: Computer graphics is widely used in movies (VFX, CGI), video games, and animated cartoons. Studios like Pixar and DreamWorks rely heavily on CG for creating stunning visual content.
  • CAD/CAM: Computer Aided Design is used extensively in engineering fields for designing mechanical parts, buildings, bridges, and circuit boards. CAM uses these designs for automated manufacturing.
  • Medical Imaging: CG enables visualization of MRI scans, CT scans, X-ray images, and 3D anatomical models of the human body, helping doctors in accurate diagnosis and surgical planning.
  • Education & Training: Realistic simulations like flight simulators for pilot training, virtual surgical training for medical students, and interactive virtual labs make learning safer and more effective.
  • Geographical Information Systems (GIS): CG is used in creating digital maps, satellite imagery visualization, terrain modeling, and navigation systems like Google Maps and GPS applications.
